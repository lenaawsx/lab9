Что делает useEffect в приложении?

Первый useEffect загружает список пород при первом рендере страницы.

Второй useEffect срабатывает при выборе породы и загружает соответствующие картинки.

useEffect позволяет выполнять побочные эффекты (запросы к API) только тогда, когда это нужно, не создавая бесконечных циклов рендеров.

Какие состояния вы использовали и зачем?

breeds — массив пород, чтобы заполнить dropdown.

selected — выбранная порода, чтобы знать, какие картинки загружать.

cats — массив изображений котиков выбранной породы.

loading — индикатор загрузки, чтобы показывать спиннер или сообщение «Загрузка…».

error — текст ошибки, чтобы отображать пользователю, если что-то пошло не так при fetch.

Что из документации API было важнее всего?

Как правильно формировать URL для получения списка пород: https://api.thecatapi.com/v1/breeds.

Как фильтровать изображения по breed_id через параметр limit.



Вот пример API-запроса для твоего проекта «Каталог котиков по породам»:

// Получение списка всех пород
const res = await fetch("https://api.thecatapi.com/v1/breeds");
const breeds = await res.json();

// Получение изображений выбранной породы (например, breed_id = "beng")
const res2 = await fetch("https://api.thecatapi.com/v1/images/search?breed_ids=beng&limit=12");
const images = await res2.json();


Первый запрос возвращает список всех пород для dropdown.

Второй — изображения котиков выбранной породы, с лимитом 12 картинок.
